关于Angular

  在分析how to use angular lightly之前， 需要对Angular的基础架构与核心模块有一定了解，这里仅对使用到的核心概念进行介绍；

angular核心概念

  • Controller（控制器）
  • Directive（指令）
  • Service （服务）

Controller

Use controllers to:

  • Set up the initial state of the $scope object.
  • Add behavior to the $scope object.

Do not use controllers to:

  • Manipulate DOM
  • Format input — Use angular form controls instead.
  • Filter output — Use angular filters instead.
  • Share code or state across controllers — Use angular services instead.
  • Manage the life-cycle of other components (for example, to create service instances).

  从上面的Dos& Don'ts可以了解到controller的作用就是扩展$scope,与服务端的controller层类似，我们不应当将复杂的业务逻辑放在controller中；

Directive

  我们平时使用Regular对directive这个概念并不陌生，Angular的directive非常强大，一个directive可以对应整个Regular的component，它相对强大之处还在于它可以与外部controller，其他directive通信；它的使用很灵活，即可以简单地对dom绑定，也可以实现像jQuery各种控件一样的页面组件；

Service

Angular services are:

  • Lazily instantiated – Angular only instantiates a service when an application component depends on it.
  • Singletons – Each component dependent on a service gets a reference to the single instance generated by the service factory.

  Service与轻量化使用Angular关系不是很大，它与服务端的Service的设计理念类似，单例的实现供需要的地方调用；
